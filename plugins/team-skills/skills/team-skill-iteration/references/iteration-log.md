# 团队 Skill 迭代记录模板

## 使用方式

当团队成员在使用任意 team skill 时发现触发、流程、规则、输出、资源或验证问题，各自生成或追加自己的迭代记录文件。评审时再汇总多名成员的记录，统一判断是否升级具体 skill。不要因为单次问题直接修改对应 skill，除非用户明确要求立即修正文档错误。

本文件是记录格式模板，不要求所有成员直接写入同一个共享文件。若用户没有指定记录文件，可在当前任务仓库或团队约定位置创建一份同格式记录文件，并在交付中说明路径。

## 状态

- `待评审`：已记录，等待团队判断是否采纳。
- `已采纳`：评审同意优化，等待实际修改。
- `已拒绝`：评审后决定不修改 skill，保留原因。
- `已落地`：已完成对应 skill 更新，并关联变更。

## 优先级

- `P0`：阻塞关键任务、造成明显错误输出、涉及安全或生产风险。
- `P1`：频繁出现，影响团队复用效率或交付质量。
- `P2`：局部体验问题、说明不清、边界条件缺失。
- `P3`：低频优化建议、结构整理或可选增强。

## 记录模板

```markdown
### <YYYY-MM-DD> - <涉及 skill>

- 日期：<YYYY-MM-DD>
- 记录人或来源：<姓名 / 账号 / 任务来源 / 待补充>
- 涉及 skill：<skill name>
- 触发任务或场景：<用户任务、命中方式、上下文>
- 遇到的问题：<触发不准、步骤缺失、规则过期、输出不可复用、执行踩坑等>
- 临时解决方式：<本次如何绕过、补充判断或人工处理>
- 建议优化：<建议新增、修改、拆分或补充的规则>
- 建议落点：<SKILL.md / references/... / scripts/... / assets/... / description / 验证方式>
- 优先级：<P0 / P1 / P2 / P3>
- 状态：待评审
- 评审结论或关联变更：待评审
```

## 待评审记录

个人或单次任务的新记录追加到这里，建议按时间倒序排列。

## 已处理记录

评审后已采纳、已拒绝或已落地的记录移动到这里，并补充评审结论或关联变更。团队汇总评审文件中应保留原始来源，方便追溯。
